# Changelog

All notable changes to your project should be documented in this file after you create a repository from this template.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

This scaffold is intentionally minimal in the template repository so downstream projects start with a clean changelog. Changes to the template itself are tracked in the repository file `.github/TEMPLATE_MAINTENANCE.md`.
